# GitHub 分支、Issue、PR 操作记录文档

## 一、仓库基础信息

-   **仓库地址**：【https://github.com/KinglyHealer/WK14-40】
-   **开发规范**：`main` 为主分支，`dev` 为功能开发分支。所有优化在 `dev` 分支开发后合并至 `main`。

---

## 二、分支创建流程

1.  本地初始化仓库并绑定远程 GitHub 仓库：

    ```bash
    git init
    git remote add origin 你的仓库地址
    git checkout -b dev
    ```

2.  后续所有 V2.0 重构代码全部在 `dev` 分支编写提交，`main` 只留存稳定版本。

---

## 三、Issue 创建记录

-   **Issue 标题**：V1 代码优化整改清单
-   **Issue 内容**：
    1.  代码全在单文件，不符合模块化工程规范，需要拆 `.h` / `.c` 多文件；
    2.  代码大量魔法数字硬编码，后期修改麻烦，改用宏定义统一管理；
    3.  无自动化编译脚本，需要手写 `run.sh` 一键运行程序；
    4.  缺少测试用例文档与项目说明 README。
-   **处理方式**：在 `dev` 分支逐条开发优化，全部整改完毕后发起 PR。

---

## 四、PR 合并流程

1.  `dev` 分支全部代码开发、自测完毕；
2.  GitHub 网页端提交 Pull Request：`dev` → `main`；
3.  核对修改内容：分文件重构、宏优化、新增脚本、完善文档；
4.  审核通过，完成分支合并。

> **PR 链接**：【粘贴自己 PR 地址】

---

## 五、版本总结

`main` 分支保存最终上线 V2.0 稳定代码，`dev` 保留开发迭代记录。通过 Issue 记录缺陷、PR 完成代码合并，完整实现 Git 项目规范化管理。
