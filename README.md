# 学生成绩统计系统 V2.0 - 重构项目 README

## 一、项目简介

本项目是基于第13周单文件学生成绩统计系统 V1.0 的工程化重构升级版 V2.0。原版代码为单文件堆砌式代码，存在硬编码过多、代码冗余、结构混乱、无法复用等问题。本次重构严格遵循 C 语言模块化开发规范，完成多文件拆分、宏定义统一管理、冗余代码精简、代码健壮性优化，同时搭配 Linux 脚本自动化运行、Git 版本管理、AI 代码审查、Python 数据分析，实现小型 C 语言开源项目完整开发链路。

项目核心功能不变：支持学生人数合法性校验、学生姓名及成绩录入、成绩统计（总分/平均分/最高分/最低分）、成绩降序排序、结果可视化输出，在功能全覆盖的基础上完成实质性代码优化。

## 三、环境依赖与运行方式

### 1. 运行环境

-   **编译环境**：GCC 编译器（支持 C99 及以上）
-   **运行环境**：Linux / Windows Terminal / Windsurf 命令行
-   **辅助环境**：Python3 + Jupyter Notebook（用于数据分析）
-   **版本管理**：Git & GitHub

### 2. 项目编译运行（一键脚本）

项目已封装 Linux 一键自动化脚本，无需手动输入复杂编译指令：

```bash
# 赋予脚本执行权限
chmod +x run.sh

# 一键编译、运行、自动输出日志到output目录
./run.sh
```

### 3. 手动编译方式（备用）

```bash
# 开启严格警告模式编译，排查代码隐性BUG
gcc -Wall -Wextra src/main.c src/student.c -I./include -o app.out

# 运行程序
./app.out
```

## 四、V2.0 核心重构优化点

本次重构为实质性工程优化，并非简单格式修改，核心优化如下：

1.  **模块化分文件重构**
    打破原版单文件代码耦合问题，实现头文件与源文件分离：`include` 目录存放宏定义与函数声明，`src` 目录存放功能实现与程序入口，代码结构符合企业级 C 语言开发规范，支持功能复用与二次迭代。

2.  **消除硬编码与冗余代码**
    原版代码存在大量魔法数字（100、20、0、100），多处重复书写固定参数。重构后全部统一为宏定义，集中在 `student.h` 头文件管理，修改参数仅需改动一处，彻底消除冗余重复代码，提升代码可维护性。

3.  **代码健壮性升级**
    编译开启 `-Wall -Wextra` 严格警告模式，规避隐性语法缺陷；统一数据初始化规范，将最高分、最低分初始值改为宏定义赋值，代码可读性与规范性大幅提升。

4.  **版本可追溯**
    完整留存 V1.0 原始代码（`origin_main.c`），实现新旧版本对比，清晰记录迭代优化过程，满足项目版本管理要求。

## 六、V1.0 与 V2.0 版本对比

### 1. 旧版本（V1.0）缺陷

-   单文件开发，代码高度耦合，函数声明、实现、主逻辑混杂，不符合工程规范
-   大量硬编码魔法数字，参数修改繁琐，极易出现代码不一致问题
-   代码冗余严重，重复定义校验区间、固定数值，维护成本高
-   无标准化编译校验，存在隐性代码风险

### 2. 新版本（V2.0）优势

-   模块化分文件架构，职责清晰，代码可复用、可拓展
-   宏定义统一管理常量，一键修改全局参数，彻底消除冗余代码
-   严格编译警告检测，代码稳定性、健壮性显著提升
-   搭配自动化脚本、版本管理、数据分析，完整贴合小型开源项目开发流程
